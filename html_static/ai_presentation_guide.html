Core Principles

AI 발표자료의 3가지 핵심 원칙

🎯

목적을 먼저 말해라

"발표자료 만들어줘"가 아니라
"누구에게, 무엇을, 왜"를 먼저 설명한다.

나쁜 예: 발표자료 만들어줘
좋은 예: 개발팀 리뷰에서 BMS 검증 도구를 소개할 10페이지 HTML 슬라이드 만들어줘
🧩

콘텐츠를 분리해라

디자인 요청과 내용 요청을 섞지 않는다.
내용 먼저 → 디자인 나중의 순서를 지킨다.

1단계: 슬라이드 구성(목차) 잡기
2단계: 각 슬라이드 내용 채우기
3단계: 디자인/스타일 적용
🔄

반복해서 개선해라

한 번에 완성하려 하지 않는다.
실행 → 피드백 → 수정의 루프를 반복한다.

"4페이지 색상이 다른 페이지와 안 맞아"
"마지막 섹션을 빼고 싶어"
"Apple PPT처럼 바꿔줘"
Prompt Structure

프롬프트 기본 구조

필수 4가지 요소

1

형식 (Format)

HTML 슬라이드? PPT? 문서형? 무엇으로 만들지 명시

2

목적 (Purpose)

누구에게, 어떤 자리에서 발표할 것인지

3

내용 (Content)

슬라이드에 담을 핵심 정보, 수치, 흐름

4

스타일 (Style)

색상, 톤, 참고할 디자인 예시

스타일 지정 방법

참고 디자인 언급: "Apple PPT처럼"
색상 지정: "배경 흰색, 포인트 파란색"
기존 파일 기반: "dart_about.html과 같은 색상으로"
분위기 표현: "깔끔하고 전문적으로"

프롬프트 템플릿

# 이 구조로 요청하면 됩니다 [형식] HTML 슬라이드(키보드 ← → 이동)로 만들어줘 [목적] 개발팀 리뷰에서 {프로젝트명}을 발표하기 위한 자료야 [내용] 포함할 내용: - 배경: {왜 만들었나} - 핵심 기능: {무엇을 하나} - 기술 스택: {어떻게 만들었나} - 결과/성과: {어떤 효과가 있나} [스타일] Apple 발표자료처럼 깔끔하게, 다크 테마, 총 8페이지
💡 팁: 처음엔 구조(목차)만 먼저 잡고, "이 구조로 맞으면 각 슬라이드 내용 채워줘"라고 2단계로 진행하면 더 정확한 결과가 나옵니다.
Code Project Presentation

코드 프로젝트 발표자료 만들기

추천 슬라이드 구성 (7~10페이지)

1 Title — 프로젝트명, 한줄 설명, 핵심 수치
2 Background — 왜 필요했나 (문제 정의)
3 Architecture — 전체 구조, 데이터 흐름
4 Key Features — 핵심 기능 2~3개 상세
5 Tech Stack — 기술 선택 이유 포함
6 Results — 성과, 수치, Before/After
7 Summary — 핵심 가치 요약, 다음 단계

실제 사용 프롬프트 예시 (이번 DART)

# 실제로 사용한 프롬프트 흐름 Step 1. 구조 잡기 "DART BMS 검증 도구 발표자료를 HTML 슬라이드로 만들어줘. 키보드 이동, 다크 테마. 포함: 배경/파이프라인/검증항목/Fleet예외/ 성능최적화/신규기능/요약" Step 2. 디자인 개선 "Apple PPT처럼 화이트 테마로 바꿔줘. dart_about.html과 같은 색상 써줘" Step 3. 내용 수정 "4페이지 출력 형식에서 XLSX 색상코딩을 조건부 서식으로 바꿔줘" Step 4. 기능 추가 "마지막에 웹 인터페이스 슬라이드 추가해줘"
핵심: README나 코드 구조를 붙여넣으면 AI가 내용을 알아서 정리합니다. 코드/문서가 있다면 그것부터 공유하세요.

일반 발표 프롬프트 구조

# 일반 발표자료 요청 템플릿 발표 주제: {주제} 청중: {누구에게 발표하나} 예) 팀장 보고, 전사 공유, 고객 제안 목표: {이 발표로 무엇을 얻고 싶나} 예) 승인, 공감, 이해, 의사결정 핵심 메시지: {한 문장으로} 포함할 내용: - {항목 1} - {항목 2} - {항목 3} 분량: {N페이지, N분 발표} 스타일: {참고 디자인 또는 분위기}

상황별 추천 구성

📊 데이터/분석 보고
배경 → 데이터/방법 → 발견사항 → 시사점 → 제언
💡 아이디어/제안
문제 정의 → 현황 → 제안 → 기대효과 → 실행계획
📈 성과 공유
목표 → 실행내용 → 결과(수치) → 배운 점 → 다음 단계
🎓 교육/공유
왜 중요한가 → 핵심 개념 3가지 → 실습/예시 → 정리
💡 스토리텔링 팁: "이 발표의 핵심 메시지는 한 문장으로 X다"를 먼저 AI에게 알려주면, 전체 흐름이 그 메시지를 향해 자연스럽게 구성됩니다.

반복 개선 — 대화로 완성하기

자주 쓰는 수정 요청 패턴

디자인 변경:
"3페이지 카드 배경색을 더 밝게 바꿔줘"
내용 수정:
"2페이지 세 번째 항목 문구를 더 간결하게"
구조 변경:
"5페이지와 6페이지 순서를 바꿔줘"
추가/삭제:
"마지막에 Q&A 슬라이드 하나 추가해줘"
전체 스타일 교체:
"전체를 다크 테마에서 화이트 테마로 바꿔줘"

좋은 피드백 주는 법

✗ 모호한 피드백:
"좀 더 좋게 만들어줘"
✓ 구체적 피드백:
"3페이지 타이틀 폰트가 너무 작아. 28px → 36px로 키워줘"
✗ 여러 요청 한번에:
"디자인도 바꾸고 내용도 고치고 페이지도 추가해줘"
✓ 하나씩 요청:
디자인 수정 → 확인 → 내용 수정 → 확인 순서로
실전 팁: 브라우저에서 직접 열어보고, 마음에 안 드는 부분을 스크린샷 찍어서 "이 부분을 이렇게 바꿔줘"라고 하면 정확도가 훨씬 높아집니다.

발표자료 만들기 체크리스트

이 순서대로 진행하면 대부분의 발표자료를 30분 안에 완성할 수 있습니다

📋
1단계 · 준비 (5분)
☐ 청중이 누구인지 정의
☐ 핵심 메시지 한 문장 작성
☐ 포함할 내용 목록 작성
☐ 참고할 디자인 결정
⚡
2단계 · 생성 (10분)
☐ 템플릿 프롬프트로 구조 요청
☐ 구조 확인 후 내용 채우기 요청
☐ 브라우저에서 바로 확인
☐ 전체 흐름 검토
✨
3단계 · 다듬기 (15분)
☐ 디자인/색상 피드백
☐ 내용 정확도 검토
☐ 불필요한 슬라이드 제거
☐ 발표 연습 후 최종 수정
"AI에게 완성된 결과물을 요청하지 말고, 함께 만들어가는 과정으로 접근하세요.
좋은 발표자료는 좋은 대화에서 나옵니다."
